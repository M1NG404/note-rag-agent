# 阶段 7：向量存储与检索

## 本阶段要解决什么

前面你已经知道文本可以变成向量，文档可以加载和切分。本阶段把这些能力合在一起：把文档向量存进向量库，再根据用户问题做相似度检索。

对应脚本：

- `26内存向量存储.py`
- `27外部向量持久化存储.py`

## 向量库是什么

向量库负责保存：

- 文档原文。
- 文档元数据。
- 文档对应的向量。

查询时，它会把用户问题也转成向量，然后找出最相似的文档。

## 内存向量库

脚本 `26`：

```python
vector_store = InMemoryVectorStore(
    embedding=DashScopeEmbeddings()
)
```

内存向量库适合学习和临时测试。程序结束后，数据通常不会持久保存。

脚本流程：

1. 创建向量库。
2. 用 `CSVLoader` 加载 `data/info.csv`。
3. 调用 `add_documents()` 写入文档。
4. 删除部分文档。
5. 调用 `similarity_search()` 检索。

## 添加文档

```python
vector_store.add_documents(
    documents=documents,
    ids=["id"+str(i) for i in range(1, len(documents)+1)]
)
```

这里的 `documents` 是 `CSVLoader` 加载出来的 `list[Document]`。

`ids` 是给每条文档指定唯一标识。后续删除、更新时可以通过 id 操作。

## 删除文档

```python
vector_store.delete(["id1", "id2"])
```

删除后，检索结果中就不应该再出现这些 id 对应的文档。

## 相似度检索

```python
result = vector_store.similarity_search(
    "瑞达法",
    3
)
```

第二个参数 `3` 表示返回最相关的 3 条文档。

它内部发生了几步：

```text
用户问题
-> Embedding
-> 和库中向量计算相似度
-> 返回 top k 文档
```

## Chroma 持久化向量库

脚本 `27`：

```python
vector_store = Chroma(
    collection_name="test",
    embedding_function=DashScopeEmbeddings(),
    persist_directory="./chroma_db"
)
```

参数说明：

- `collection_name`：集合名称，类似数据库中的表名。
- `embedding_function`：用于把文本转成向量的模型。
- `persist_directory`：向量库保存目录。

Chroma 的优势是数据可以持久化。你第一次写入文档后，后续再运行脚本可以直接检索已有数据。

## metadata 过滤

脚本 `27` 检索时加了过滤条件：

```python
result = vector_store.similarity_search(
    "Python是不是简单易学呀",
    3,
    filter={"source": "黑马程序员"}
)
```

这表示只在 `source` 为 `黑马程序员` 的文档中检索。

metadata 过滤很实用，例如：

- 只检索某个部门的文档。
- 只检索某一年份的资料。
- 只检索某个来源的数据。

## 内存库和持久化库对比

| 类型 | 数据是否持久化 | 适合场景 |
| --- | --- | --- |
| InMemoryVectorStore | 否 | 学习、临时实验、单元测试 |
| Chroma | 是 | 本地 RAG 原型、小型项目 |

## 易错点

- 向量库写入和检索要使用兼容的 Embedding 模型。
- 重新换 Embedding 模型后，旧向量库最好重建。
- 删除文档依赖 id，建议给文档设置稳定 id。
- metadata 过滤字段必须真实存在，否则可能检索不到结果。
