# 阶段 5：会话记忆

## 本阶段要解决什么

普通模型调用是“无记忆”的。你每次调用模型，如果不把历史对话传进去，模型就不知道前面发生过什么。本阶段学习如何让链自动携带会话历史。

对应脚本：

- `20临时会话记忆.py`
- `21长期会话记忆.py`

## 会话记忆的本质

会话记忆不是模型真的永久记住了你，而是程序把历史消息保存起来，在下一次调用时重新放进 prompt。

流程是：

```text
用户输入
-> 读取当前 session 的历史消息
-> 组装 prompt
-> 调用模型
-> 把本轮用户输入和模型回复写回历史
```

## MessagesPlaceholder

两个脚本都使用了：

```python
prompt = ChatPromptTemplate.from_messages(
    [
        ("system", "你需要根据会话历史回应用户问题。对话历史："),
        MessagesPlaceholder("chat_history"),
        ("human", "请回答如下问题：{input}")
    ]
)
```

`MessagesPlaceholder("chat_history")` 是历史消息插入的位置。后面 `RunnableWithMessageHistory` 会自动把历史消息放到这里。

## 临时记忆

脚本 `20` 使用：

```python
from langchain_core.chat_history import InMemoryChatMessageHistory
```

并创建了一个字典：

```python
store = {}
```

`store` 的结构可以理解为：

```python
{
    "user_001": InMemoryChatMessageHistory(...)
}
```

`get_history(session_id)` 的作用是：根据会话 ID 找到对应历史，如果没有就新建。

```python
def get_history(session_id):
    if session_id not in store:
        store[session_id] = InMemoryChatMessageHistory()

    return store[session_id]
```

临时记忆只存在于程序运行期间。程序关闭后，`store` 就没了。

## RunnableWithMessageHistory

这是本阶段最重要的组件：

```python
conversation_chain = RunnableWithMessageHistory(
    base_chain,
    get_history,
    input_messages_key="input",
    history_messages_key="chat_history"
)
```

参数含义：

- `base_chain`：原始链，不带记忆能力。
- `get_history`：告诉 LangChain 如何根据 session_id 获取历史。
- `input_messages_key="input"`：用户输入字段叫什么。
- `history_messages_key="chat_history"`：prompt 中历史占位符叫什么。

调用时还要传配置：

```python
session_config = {
    "configurable": {
        "session_id": "user_001"
    }
}
```

这个 session_id 决定当前调用属于哪一个用户或哪一段会话。

## 长期记忆

脚本 `21` 自定义了 `FileChatMessageHistory`：

```python
class FileChatMessageHistory(BaseChatMessageHistory):
```

它把消息保存到本地文件：

```python
self.file_path = os.path.join(self.storage_path, self.session_id)
```

核心方法有三个：

- `add_messages()`：新增消息并写入文件。
- `messages`：从文件读取历史消息。
- `clear()`：清空历史。

保存时先把消息对象转成字典：

```python
message_to_dict(message)
```

读取时再把字典转回消息对象：

```python
messages_from_dict(messages_data)
```

这样程序重启后仍能读取之前的历史。

## 临时记忆和长期记忆对比

| 类型 | 保存位置 | 程序关闭后是否保留 | 适合场景 |
| --- | --- | --- | --- |
| 临时记忆 | 内存 | 否 | 快速测试、单次会话 |
| 长期记忆 | 文件或数据库 | 是 | 用户历史、长期助手 |

## 易错点

- `session_id` 不同，读取到的历史就不同。
- `input_messages_key` 必须和调用时传入的字段一致。
- `history_messages_key` 必须和 `MessagesPlaceholder` 名字一致。
- 长期记忆保存用户对话时要注意隐私和清理机制。
