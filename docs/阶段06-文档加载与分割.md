# 阶段 6：文档加载与分割

## 本阶段要解决什么

RAG 的知识来源通常不是手写字符串，而是 CSV、JSON、PDF、TXT、网页、数据库等真实资料。本阶段学习如何把不同格式的数据加载成 LangChain 统一的 Document 对象，并进一步切分成适合检索的小片段。

对应脚本：

- `22CSVLoader的使用.py`
- `23JSONLoader的使用.py`
- `24PyPDFLoader的使用.py`
- `25TextLoader和文档分割器.py`

## Document 对象

LangChain 加载文档后，通常返回：

```python
list[Document]
```

每个 `Document` 主要包含两部分：

- `page_content`：正文内容。
- `metadata`：来源、页码、行号等元数据。

你可以把 Document 理解为 RAG 里的标准资料格式。

## CSVLoader

脚本 `22`：

```python
loader = CSVLoader(
    file_path="./data/stu.csv",
    csv_args={
        "delimiter": ",",
        "quotechar": '"',
        "fieldnames": ['name', 'age', 'gender', '爱好']
    },
    encoding="utf-8"
)
```

参数说明：

- `file_path`：CSV 文件路径。
- `delimiter`：分隔符，常见是逗号。
- `quotechar`：当字段里也有逗号时，用什么符号包住该字段。
- `fieldnames`：如果原始 CSV 没有表头，可以手动指定字段名。
- `encoding`：文件编码。

脚本中使用了：

```python
for document in loader.lazy_load():
    print(document)
```

`lazy_load()` 是懒加载，适合大文件；`load()` 是一次性全部加载，适合小文件。

## JSONLoader

脚本 `23`：

```python
loader = JSONLoader(
    file_path="./data/stu_json_lines.json",
    jq_schema=".name",
    text_content=False,
    json_lines=True
)
```

参数说明：

- `jq_schema=".name"`：抽取每条 JSON 里的 `name` 字段。
- `text_content=False`：抽取的内容不是普通字符串时使用。
- `json_lines=True`：文件是 JSON Lines 格式，每一行都是一个独立 JSON。

JSONLoader 的关键是写对 `jq_schema`。如果你要抽取整条记录，可以换成更适合的 schema。

## PyPDFLoader

脚本 `24`：

```python
loader = PyPDFLoader(
    file_path="./data/pdf2.pdf",
    mode="single",
    password="itheima"
)
```

参数说明：

- `mode="page"`：默认模式，每一页生成一个 Document。
- `mode="single"`：整个 PDF 合成一个 Document。
- `password`：PDF 有密码时需要提供。

如果后续要做检索，通常更推荐先按页加载，再进行文本切分，这样 metadata 中页码更有价值。

## TextLoader 和文本分割

脚本 `25`：

```python
loader = TextLoader("./data/Python基础语法.txt", encoding="utf-8")
docs = loader.load()
```

加载后使用递归字符分割器：

```python
splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separators=["\n\n", "\n", "。", "！", "？", ".", "!", "?", " ", ""],
    length_function=len,
)
```

参数说明：

- `chunk_size=500`：每段最多 500 个字符。
- `chunk_overlap=50`：相邻片段重叠 50 个字符。
- `separators`：优先按哪些符号切分。
- `length_function=len`：用什么函数计算长度。

## 为什么要分割文档

如果文档太长，直接送给模型会有几个问题：

- 超过模型上下文限制。
- 检索结果太粗，相关信息被无关内容稀释。
- 成本更高，速度更慢。

合理切分后，向量库检索到的是更精确的小片段。

## chunk_size 和 chunk_overlap 怎么选

没有固定答案，可以按任务调整：

- FAQ、短知识点：`chunk_size` 可以小一些。
- 教程、法规、论文：`chunk_size` 可以大一些。
- 句子之间强依赖：适当增加 `chunk_overlap`。

当前脚本的 `500/50` 是学习阶段比较好理解的配置。

## 易错点

- CSV 有表头时不一定要传 `fieldnames`。
- 文件编码不对会导致乱码。
- PDF 提取效果取决于 PDF 本身，有些扫描版 PDF 需要 OCR。
- 切分太碎会丢上下文，切分太大会降低检索精度。
