## 你需要先理解的概念

### LLM 和 Chat Model

LangChain 中常见两类模型接口：

- LLM：输入通常是一段普通字符串，输出也是文本。
- Chat Model：输入通常是多条消息，消息带有角色，例如 `system`、`human`、`ai`。

脚本 `02` 使用的是 `Tongyi`：

```python
from langchain_community.llms.tongyi import Tongyi

model = Tongyi(model="qwen-max")
res = model.invoke(input="你是谁呀能做什么？")
```

这里的 `qwen-max` 被当作普通大语言模型使用。你直接给它一个字符串，它直接返回一段文本。

脚本 `05` 使用的是 `ChatTongyi`：

```python
from langchain_community.chat_models.tongyi import ChatTongyi

model = ChatTongyi(model="qwen3-max")
```

聊天模型更适合多轮对话，因为它能接收带角色的消息列表。

## 云端模型和本地模型

脚本 `02` 调用阿里云通义千问，脚本 `03` 调用 Ollama 本地模型：

```python
from langchain_ollama import OllamaLLM

model = OllamaLLM(model="qwen3:4b")
res = model.invoke(input="你是谁呀能做什么？")
```

两者学习重点不同：

- 云端模型：通常效果更强，但需要 API Key、网络和服务额度。
- 本地模型：数据留在本机，适合学习和调试，但需要提前安装 Ollama 并拉取模型。

如果本地脚本报错，优先检查：

- Ollama 服务是否启动。
- `qwen3:4b` 是否已经拉取到本地。
- 模型名称是否和本机 `ollama list` 中显示的一致。

## invoke 和 stream

### invoke

`invoke()` 表示一次性调用。你把输入交给模型，等模型完整生成后再返回结果。

适合：

- 简短问答。
- 脚本测试。
- 不关心逐字输出体验的场景。

### stream

脚本 `04` 演示了流式输出：

```python
res = model.stream(input="你是谁呀能做什么？")

for chunk in res:
    print(chunk, end="", flush=True)
```

`stream()` 会边生成边返回片段。聊天产品里常见的“逐字出现”效果就是这个思路。

如果是 Chat Model，流式返回的 chunk 通常需要通过 `.content` 取文本：

```python
for chunk in res:
    print(chunk.content, end="", flush=True)
```

## 消息角色

脚本 `05` 使用了完整消息类：

```python
messages = [
    SystemMessage(content="你是一个边塞诗人。"),
    HumanMessage(content="写一首唐诗"),
    AIMessage(content="锄禾日当午，汗滴禾下土，谁知盘中餐，粒粒皆辛苦。"),
    HumanMessage(content="按照你上一个回复的格式，在写一首唐诗。")
]
```

每种消息都有不同含义：

- `SystemMessage`：设定模型身份、规则、边界。
- `HumanMessage`：用户输入。
- `AIMessage`：模型历史回复。

脚本 `07` 展示了简写形式：

```python
messages = [
    ("system", "你是一个边塞诗人。"),
    ("human", "写一首唐诗。"),
    ("ai", "锄禾日当午，汗滴禾下土，谁知盘中餐，粒粒皆辛苦。"),
    ("human", "按照你上一个回复的格式，在写一首唐诗。")
]
```

这个写法更短，在后面的 `ChatPromptTemplate` 里会经常看到。

## 代码运行顺序建议

1. 先运行 `02`，确认你能调通云端普通模型。
2. 再运行 `03`，确认你能调通本地普通模型。
3. 运行 `04`，观察 `invoke()` 和 `stream()` 的体验差异。
4. 运行 `05` 和 `06`，理解聊天模型的消息列表。
5. 运行 `07`，记住消息可以用元组简写。
