# 阶段 3：提示词模板

## 本阶段要解决什么

当问题变复杂时，不能每次都手写完整提示词。提示词模板可以把固定规则和动态变量分开，让代码更清晰，也更容易复用。

对应脚本：

- `10通用提示词模板.py`
- `11FewShot提示词模板.py`
- `12模板类的format和invoke方法.py`
- `13ChatPromptTemplate的使用.py`

## PromptTemplate

脚本 `10` 使用普通提示词模板：

```python
prompt_template = PromptTemplate.from_template(
    "我的邻居姓{lastname}, 刚生了{gender}, 你帮我起个名字，简单回答。"
)
```

模板中的 `{lastname}` 和 `{gender}` 是变量。调用时传入字典：

```python
res = chain.invoke(input={"lastname": "张", "gender": "女儿"})
```

模板会被渲染成：

```text
我的邻居姓张, 刚生了女儿, 你帮我起个名字，简单回答。
```

## format 和 invoke

脚本 `12` 对比了两个方法：

```python
res = template.format(lastname="张大明", hobby="钓鱼")
print(res, type(res))

res2 = template.invoke({"lastname": "周杰轮", "hobby": "唱歌"})
print(res2, type(res2))
```

区别：

- `format()`：直接返回字符串。
- `invoke()`：返回 LangChain 的 PromptValue 对象。

如果你只是想看最终提示词文本，用 `format()` 很直观。如果你要把模板接入 Chain，通常用 `invoke()` 或直接使用 `prompt | model`。

## Chain 写法

脚本 `10` 中有这一行：

```python
chain = prompt_template | model
```

这表示把提示词模板的输出交给模型：

```text
dict -> PromptTemplate -> prompt text -> Model -> model output
```

这就是 LangChain Expression Language 的基础写法。后面的 RAG 链也是不断扩展这个思路。

## FewShotPromptTemplate

脚本 `11` 演示 Few-shot：

```python
example_template = PromptTemplate.from_template("单词：{word}, 反义词：{antonym}")

examples_data = [
    {"word": "大", "antonym": "小"},
    {"word": "上", "antonym": "下"},
]
```

Few-shot 的意思是：先给模型几个示例，再让模型照着示例完成新任务。

最终提示词由三部分组成：

- `prefix`：告诉模型任务是什么。
- `examples`：给模型示范输入输出格式。
- `suffix`：提出真正的问题。

这类提示词适合：

- 要求模型遵守固定格式。
- 希望模型模仿示例风格。
- 简单分类、转换、抽取任务。

## ChatPromptTemplate

脚本 `13` 使用聊天提示词模板：

```python
chat_prompt_template = ChatPromptTemplate.from_messages(
    [
        ("system", "你是一个边塞诗人，可以作诗。"),
        MessagesPlaceholder("history"),
        ("human", "请再来一首唐诗"),
    ]
)
```

这里重点是 `MessagesPlaceholder("history")`。它表示这个位置将来要插入一组历史消息。

示例历史：

```python
history_data = [
    ("human", "你来写一个唐诗"),
    ("ai", "床前明月光，疑是地上霜，举头望明月，低头思故乡"),
]
```

最终提示词会把 system、历史消息、最新 human 问题组合在一起。

## 提示词模板和 RAG 的关系

RAG 不是只做检索，还要把检索到的内容组织进提示词。后面你会看到类似这种模板：

```python
("system", "以我提供的已知参考资料为主，回答用户问题。参考资料:{context}")
("user", "用户提问：{input}")
```

这里的 `{context}` 就是检索到的文档内容，`{input}` 就是用户问题。

## 易错点

- 模板变量名必须和传入字典的 key 一致。
- ChatPromptTemplate 更适合聊天模型，PromptTemplate 更适合普通文本模型。
- Few-shot 示例不要太多，否则会占用上下文窗口。
- 提示词模板只负责组织输入，不负责保证模型一定正确。
